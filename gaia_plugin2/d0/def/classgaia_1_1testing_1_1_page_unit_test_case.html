Detailed Description

template<class TesPage, class TestFixture>
class gaia::testing::PageUnitTestCase< TesPage, TestFixture >

This class provides isolated testing of a single Page.

The Page under test will be created with minimal connection to the system infrastructure, and you can inject mocked or wrappered versions of many of Page's dependencies. Most of the work is handled automatically here by SetUp and TearDown.

If you prefer a functional test, see gaia::testing::PageInstrumentationTestCase.

It must be noted that, as a true unit test, your Page will not be running in the normal system and will not participate in the normal interactions with other Pages. The following methods should not be called in this configuration - most of them will throw exceptions:

  • gaia::core::Page#getCallingPage()
  • gaia::core::Page#getCallingPackage()
  • gaia::core::Page#isTaskRoot()
  • gaia::core::Page#moveTaskToBack(bool)

The following methods may be called but will not do anything. For test purposes, you can use the methods getStartedPageDemand() and getStartedPageRequest() to inspect the parameters that they were called with.

  • gaia::core::Page#startPage(Demand*)
  • gaia::core::Page#startPageForResult(Demand*, int)

The following methods may be called but will not do anything. For test purposes, you can use the methods isFinishCalled() and getFinishedPageRequest() to inspect the parameters that they were called with.

  • gaia::core::Page#finish()
  • gaia::core::Page#finishPage(int requestCode)

Inherits gaia::testing::PageTestCase.

template<class TesPage , class TestFixture >
TesPage * gaia::testing::PageUnitTestCase< TesPage, TestFixture >::startPage ( Demand *  demand,
Persistence *  savedInstanceState 
) [protected]

Start the Page under test, in the same way as if it was started by gaia::core::Context#startPage(Demand*), providing the arguments it supplied.

When you use this method to start the Page, it will automatically be stopped by TearDown.

This method will call onCreate(), but if you wish to further exercise Page life cycle methods, you must call them yourself from your test case.

Do not call from your SetUp() method. You must call this method from each of your test methods.

Parameters:
demandThe demand as if supplied to gaia::core::Context#startPage(Demand*).
savedInstanceStateThe instance state, if you are simulating this part of the life cycle. Typically null.
Returns:
Returns the Page that was created
