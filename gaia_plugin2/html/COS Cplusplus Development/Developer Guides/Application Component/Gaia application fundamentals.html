Gaia Application Fundamentals

Gaia Applications are written in the C++ programming language. The COS sdk compile the code--along with any data and resource file--into COS package, an archive file with an .apk suffix.

Once installed on a device, each COS application lives in its own security sandbox:

  • The COS operating system is a multi-user Linux system in which each application is a different user.
  • By default, the system assigns each application a unique Linux user ID (the ID is used only by the system and is unknown to the application). The system sets permissions for all the files in an application so that only the user ID assigned to that application can access them.
  • By default, every application runs in its own Linux process. COS starts the process when any of the application's components need to be executed, then shuts down the process when it's no longer needed or when the system must recover memory for other applications.

In this way, the COS system implements the principle of least privilege. That is, each application, by default, has access only to the components that it requires to do its work and no more. This creates a very secure environment in which an application cannot access parts of the system for which it is not given permission.
