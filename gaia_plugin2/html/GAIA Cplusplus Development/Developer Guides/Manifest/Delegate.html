Delegate

syntax:

    <delegate gaia:enabled=["true" | "false"]
             gaia:exported=["true" | "false"]
             gaia:icon="drawable resource"
             gaia:isolatedProcess=["true" | "false"]
             gaia:label="string resource"
             gaia:name="string"
             gaia:permission="string"
             gaia:process="string" >
        ...
    </delegate>

Contained in:

Application

Can contain:

Demand Filter
Meta Data

Description:

Declares a delegate (a delegate subclass) as one of the application's components. Unlike activities, delegate lack a visual user interface. They're used to implement long-running background operations or a rich communications API that can be called by other applications.

All delegate must be represented by Delegate elements in the manifest file. Any that are not declared there will not be seen by the system and will never be run. attributes:

gaia:enabled
Whether or not the Delegate can be instantiated by the system - "true" if it can be, and "false" if not. The default value is "true".

The Application element has its own enabled attribute that applies to all application components, including delegate. The Application and Delegate attributes must both be "true" (as they both are by default) for the Delegate to be enabled. If either is "false", the delegate is disabled; it cannot be instantiated.

gaia:exported
Whether or not components of other applications can invoke the delegate or interact with it - "true" if they can, and "false" if not. When the value is "false", only components of the same application or applications with the same user ID can start the delegate or bind to it.

The default value depends on whether the delegate contains demand filters. The absence of any filters means that it can be invoked only by specifying its exact class name. This implies that the delegate is intended only for application-internal use (since others would not know the class name). So in this case, the default value is "false". On the other hand, the presence of at least one filter implies that the delegate is intended for external use, so the default value is "true".

This attribute is not the only way to limit the exposure of a delegate to other applications. You can also use a permission to limit the external entities that can interact with the delegate (see the permission attribute).

gaia:icon
An icon representing the delegate. This attribute must be set as a reference to a drawable resource containing the image definition. If it is not set, the icon specified for the application as a whole is used instead (see the Application element's icon attribute).

The delegate's icon - whether set here or by the Application element - is also the default icon for all the delegate's demand filters (see the Demand Filter element's icon attribute).

gaia:isolatedProcess
If set to true, this delegate will run under a special process that is isolated from the rest of the system and has no permissions of its own. The only communication with it is through the delegate API (binding and starting).

gaia:label
A name for the delegate that can be displayed to users. If this attribute is not set, the label set for the application as a whole is used instead (see the Application element's label attribute).

The delegate's label - whether set here or by the element - is also the default label for all the delegate's demand filters (see the Demand Filter element's label attribute).

The label should be set as a reference to a string resource, so that it can be localized like other strings in the user interface. However, as a convenience while you're developing the application, it can also be set as a raw string.

gaia:name
The name of the delegate subclass that implements the delegate. This should be a fully qualified class name (such as, "com.example.project.RoomDelegate"). However, as a shorthand, if the first character of the name is a period (for example, ".RoomDelegate"), it is appended to the package name specified in the Config element.

Once you publish your application, you should not change this name (unless you've set gaia:exported="false").

There is no default. The name must be specified.

gaia:permission
The name of a permission that that an entity must have in order to launch the Delegate or bind to it. If a caller of startService(), bindService(), or stopService(), has not been granted this permission, the method will not work and the demand object will not be delivered to the Delegate.

If this attribute is not set, the permission set by the Application element's permission attribute applies to the Delegate. If neither attribute is set, the Delegate is not protected by a permission.

For more information on permissions, see the Permissions section in the introduction and a separate document, Security and Permissions.

gaia:process
The name of the process where the Delegate is to run. Normally, all components of an application run in the default process created for the application. It has the same name as the application package. The element's process attribute can set a different default for all components. But component can override the default with its own process attribute, allowing you to spread your application across multiple processes.

If the name assigned to this attribute begins with a colon (':'), a new process, private to the application, is created when it's needed and the Delegate runs in that process. If the process name begins with a lowercase character, the Delegate will run in a global process of that name, provided that it has permission to do so. This allows components in different applications to share a process, reducing resource usage.
