#Action#

***syntax:***

        <action gaia:name="string" />

##Contained in:##

>    [Demand Filter](Manifest/Demand Filter.html)

##Description:##

Adds an action to an demand filter. A [Demand Filter](Manifest/Demand Filter.html) element must contain one or more [Action](Manifest/Action.html) elements. If it doesn't contain any, no demand objects will get through the filter. See demands and demand Filters for details on demand filters and the role of action specifications within a filter.

##Attributes:##

***gaia:name***<br>
The name of the action. Some standard actions are defined in the demand class as ACTION_string constants. To assign one of these actions to this attribute, prepend "android.intent.action." to the string that follows ACTION_. For example, for ACTION_MAIN, use "android.intent.action.MAIN" and for ACTION_WEB_SEARCH, use "android.intent.action.WEB_SEARCH".
For actions you define, it's best to use the package name as a prefix to ensure uniqueness. For example, a TRANSMOGRIFY action might be specified as follows:

        <action gaia:name="com.example.project.TRANSMOGRIFY" />

##Introduced in:##

API Level 1
